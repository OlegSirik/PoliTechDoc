# Базовый сценарий, для работы через АПИ, без создания юзерских учеток.
Тут нужны примеры вызовов API.    
Клиента будем тестирвоать по на правильный порядок вызова АПИ методов.    

# Установка системы
1. Создать тенант SYS ( сейчас VSK )  
2. Есть acc_account для тенанта SYS c типом TENANT
3. Есть админская учетка system/manager
4. Могу зайти с ней в приложение, добавить другого SYS_ADMIN и удалить дефолтный

# Добавить тенанты 
1. Предусловие -  у меня есть sys_admin учетка. я залогинен и получил токен.
2. Завожу тенат VSK
3. Есть тенат VSK в acc_account с ролью TENANT
4. Могу добавить учетку для TNT_ADMIN
5. Добавляю себя.
6. Тоже для тенанта msg
   
# Добавить клиента
1. У меня есть учетка для тенанта VSK с ролью TNT_ADMIN
2. Переключаюсь на этот account_id
3. Добавляю клиента 
4. Есть запись в acc_clients, и в acc_accounts с ролью CLIENT, paretn_id = tenant.vsk
5. Добавляю учетку для client с ролью client_admin
6. Даю себе права на эту группу.
7. Добавить account для продажи, роль SALE, paretn = CLIENT
8. Вернуться в клиента, обновить его, установив дефолтный sale account
9. Может сделать при создании CLEINT признак, что это клиент без юзерских учеток, и создавать дефолтный портфель (sale account) срузу

# Добавить продукт
1. Залогинится как tnt_admin
2. Добавить клиенту продуктовые права
3. Права на продукты для клиента может добавлять только tnt_admin
