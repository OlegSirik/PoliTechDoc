# tenants
## Entity
На текщий момент содержит минимальный набор параметров. Далее будет расширятся по необходимости.

### code
код тенанта.   
уникальный,   
всегда в нижнем регистре,
только латинские буквы и цифры.

Используется в урл как для фронта так и для бэка, поэтому желательно создавать не длинные коды.

### name
Наименование, понятное, читабельное

### is_deleted
Признак временного или полного удаления тенанта.  
Если is_deleted = true, то все действия с этим разделом запрещены.

## ФТ

При начальной установке системы создается тенант - sys. Он содержит списки sys_admin, и нужен для начальной настройки системы.
Далее создаются тенанты, например prod & test со своими tnt_admin, и далее работа ведется уже в этих тенантах.
