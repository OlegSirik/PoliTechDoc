## Line-of-bussines 
Lob - максимально полная структура входящего запроса для продуктовой группы. Только с типом IN, плюс MAGIC (производная от IN), но их можно сократить.  
Категории vars. Префиксы -   
1. pl - договор
Возможно additional_property
2. ph - тип страхователя
Возможно additional_property
3. io - объект страхования
Не валидируется
4. co - покрытия ( страх. сумма, премия , франшиза )
Фиксированная структура. В LoB не добавляется.

## Версия продукта 
Все с LoB. Можно удалить все что не нужно. Остаток формирует входящий запрос на создание продукта.
По полям с типом IN строится форма ввода.
Ограничение по покрытиям. Только из справочника LoB. Добавление покрытия влечет добавление 3 var - premium, sumInsured, deductibleNr  
Можно добавить VAR для чего-то.

## Калькулятор 
Описывает порядок расчета премии. Привязан к версии продукта 1-1.    
vars - все с версии продукта + свои атрибуты ( Const, Coefficient, Var ).   
 
## Тип атрибута 
1. IN - приходит в запросе. Нельзя менять значение.
2. VAR - переменная участвующая в расчете. Можно изменять, может быть на старте из запроса.
3. COEFFICIENT - вычисляемое по таблице значение.
4. CONST - константа. Задается в калькуляторе. Используется для расчета.
5. MAGIC - производные от входного запроса данные. Не изменяемые. Например возраст страхователя - количество лет.

Атрибуты переменной
varCode - уникальный код атрибута. Используется в калькуляторе, генераторе ПФ и т.д.
varType - IN | VAR | CONST | MAGIC | COEFFICIENT
varDataType - NUMBER | STRING | DATE | DURATION
varName - наименование
varPath - для jsonPath для получения значения. Может быть в IN и VAR
varValue - значение. Для const - значение константы, в процессе расчета хранит значения, в справочнике значения для составления примера.
